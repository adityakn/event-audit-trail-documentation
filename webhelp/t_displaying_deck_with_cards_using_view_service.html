Displaying a Deck with Cards Using View Service

This procedure has the following requirements:
  • A REST client (such as Postman) that can execute API requests to use the View service. See View Service Database Operations for more information.

    For more information about these APIs, see the API Documentation.

  • Set up the required services for the Predix UI platform. See Setting Up Required Services for Predix UI for more information.
  • Locate the application project directory <web_app> where the following files will be installed:
    • webcomponents-lite polyfill
    • px-cardrepository from the PredixDev public GitHub organization. See the README file for more information.

The objective of this example is to show how to configure a web application with components from the px-cardrepository (in the PredixDev public GitHub organization) and View service to display a deck with two cards.

See Using View Service for more information about using the View service database to manage card and deck objects.

  1. Change to the web application project directory:
    cd <web_app>
  2. Install the webcomponents-lite polyfill by using the bower package-management utility:
    bower install webcomponentsjs --save
  3. Add the webcomponents-lite polyfill as the first line in the <head> section of your web application. For example
    <!DOCTYPE html>

    <html>

        <head>
            <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

        </head>

     </html>
  4. Install the px-card repository from the PredixDev public GitHub organization.
    bower install https://github.com/PredixDev/px-card.git --save
  5. Import the components you want to use into your web application with the following tags in your head tag. For example:
    <!DOCTYPE html>

    <html>
    
      <head>

        <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

  
        <link rel="import" href="/bower_components/px-card/px-dashboard.html"/>
        <link rel="import" href="/bower_components/px-card/px-deck.html"/>
        <link rel="import" href="/bower_components/px-card/px-card.html"/>

     </head>

 
  </html>
  6. Call the px-card repository in your web application. For example:
    <!DOCTYPE html>

    <html>
    
      <head>

        <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

  
        <link rel="import" href="/bower_components/px-card/px-dashboard.html"/>
        <link rel="import" href="/bower_components/px-card/px-deck.html"/>
        <link rel="import" href="/bower_components/px-card/px-card.html"/>

     </head>

     <body>

         <px-card></px-card>

     </body>


  </html>
  7. Create a card instance object in the View service database.. For example:
    <!DOCTYPE html>

    <html>
    
      <head>

        <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

  
        <link rel="import" href="/bower_components/px-card/px-dashboard.html"/>
        <link rel="import" href="/bower_components/px-card/px-deck.html"/>
        <link rel="import" href="/bower_components/px-card/px-card.html"/>

     </head>

     <body>

         <px-card></px-card>

         <!-- a card that can display temperature of a city -->  
         <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>

     </body>


  </html>

    Card Instance in JSON format:

    /*
Create Card in view service
[POST] /api/Cards
Request Body:
*/
[{
    "title": "San Ramon Temperature",
    "slug": "my-temperature-card",
    "attributes": {
        "postal": "94582",
        "unit": "F"
    }
},
{
    "title": "Walnut Creek Temperature",
    "slug": "my-temperature-card",
    "attributes": {
        "postal": "94595",
        "unit": "F"
    }
}]
  8. Create a deck instance object in the View service database. For example:
    <!DOCTYPE html>

    <html>
    
      <head>

        <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

  
        <link rel="import" href="/bower_components/px-card/px-dashboard.html"/>
        <link rel="import" href="/bower_components/px-card/px-deck.html"/>
        <link rel="import" href="/bower_components/px-card/px-card.html"/>

     </head>

     <body>

         <px-card></px-card>

         <!-- a card that can display temperature of a city -->  
         <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>

         <!-- a deck with 2 cards -->
         <px-deck id="deck-1">
             <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>
             <my-temperature-card id="card-3" title="Walnut Creek Temperature" postal="94595" unit="F"></my-temperature-card>
         </px-deck>

     </body>


  </html>

    Create Deck Instance in JSON format:

    /*
Create deck in view service
[POST] /api/decks
Request Body:
*/
{
    "title": "Contra Costa Deck"
}
  9. Associate card and deck instances in JSON format. For example: 
    /*
Associate Card 1,2 to Deck 1
[POST] /api/decks/1/cards/add
Request body 
*/
["1", "2"]
  10. Retrieve cards in a deck stored in JSON format. For example:
    /*
Get all cards in deck 1
[GET] /api/decks/1/cards
Response Body:
*/
{
    "id": 1,
    "title": "Contra Costa Deck",
    "cards":[
        {
            "id": 1,
            "title": "San Ramon Temperature",
            "slug": "my-temperature-card",
            "attributes": {
                "postal": "94582",
                "unit": "F"
            }
        },
        {
            "id": 2,
            "title": "Walnut Creek Temperature",
            "slug": "my-temperature-card",
            "attributes": {
                "postal": "94595",
                "unit": "F"
            }
        }
    ]
}
  11. To group a deck by categories, tag your deck in JSON format
    Tag a deck by multiple tag values
[POST] /api/decks/1/tags
Request Body:
*/
[
{"value": "contra costa"},
{"value": "california"},
{"value": "us"}
]

    Search and retrieve a deck by tag/category lookup in JSON format. For example:

    /*
Get a deck by tag/category lookup
[GET] /api/decks/tags?values=us
Response Body:
*/
[
    {
        "id": 1,
        "title": "Contra Costa Deck"
        "tags": [
            {"value": "contra costa"},
            {"value": "california"},
            {"value": "us"}
        ]
    }
]
  12. Add the px-dashboard component to consume a View service URL to get all cards of a deck stored in the View service database. The component can traverse the JSON, create DOM objects and append to an web application. Foe example: 
    <!DOCTYPE html>

    <html>
    
      <head>

        <script src="bower_components/webcomponentsjs/webcomponents.js"></script>

  
        <link rel="import" href="/bower_components/px-card/px-dashboard.html"/>
        <link rel="import" href="/bower_components/px-card/px-deck.html"/>
        <link rel="import" href="/bower_components/px-card/px-card.html"/>

     </head>

     <body>

         <px-card></px-card>

         <!-- a card that can display temperature of a city -->  
         <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>

         <!-- a deck with 2 cards -->
         <px-deck id="deck-1">
             <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>
             <my-temperature-card id="card-3" title="Walnut Creek Temperature" postal="94595" unit="F"></my-temperature-card>
         </px-deck>

         <!-- web component to add to the page -->
             <px-dashboard view-service-base-url=“/api/view-service" selected-deck-id=“1"></px-dashboard>
 
         <!-- Web component will read the json, convert into dom element and inject into the document
              <px-dashboard selected-deck="/api/decks/1?filter[include][cards]">
                  <container>
                      <px-deck id="deck-1">
                          <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>
                          <my-temperature-card id="card-3" title="Walnut Creek Temperature" postal="94595" unit="F"></my-temperature-card>
                      </px-deck>
                  </container>           
             <px-dashboard>
 
    </body>


  </html>
