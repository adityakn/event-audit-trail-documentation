Installing the Dashboard Seed

You need a Predix platform user name <your_predix_login>. and password <your_predix_password>. See Registering for a Predix Platform User Account

Set up required services to install the Dashboard Seed. Sse Setting Up Required Services to Install the Dashboard Seed.

This procedure uses the following developer tools to install the Dashboard Seed:
  • Node.js npm package-management utility. See https://www.npmjs.com/ for more information.
  • grunt command-line interface for npm. See https://www.npmjs.com/package/grunt-cli for more information.
  • grunt serve plugin, used to start a web server that runs tasks and serves files. This web server can be accessed by loading the http://localhost:9000/ page . See https://www.npmjs.com/package/grunt-serve for more information.
  • bower package manager for npm. See https://www.npmjs.com/package/bower for more information.
  1. Use the Cloud Foundry CLI to log into Cloud Foundry. 
    cf login -a <API_Endpoint>

    The <API_Endpoint> value is included in the Predix invitation email that you receive when you register for an Predix.io user platform account. Depending on your Predix.io registration, the <API_Endpoint> uses one of the following values:

    • Predix Basic

      https://api.system.aws-usw02-pr.ice.predix.io

    • Predix Select

      https://api.system.asv-pr.ice.predix.io

    For example,

    cf login -a https://api.system.aws-usw02-pr.ice.predix.io
    The command generates the following output: 
    Email> <your_predix_login> 
Password> <your_predix_password> 
Authenticating...OK 
Targeted org <your_predix_org> 
Targeted space dev     
API endpoint: https://api.system.aws-usw02-pr.ice.predix.io (API version: 2.28.0)   
User: <your_predix_login> 
Org:  <your_predix_org>  
Space: dev
  2. Clone your copy of the Dashboard Seed and start a Dashboard Seed project. 
    git clone https://github.com/PredixDev/predix-seed

    The command generates the following output: 

    Cloning into 'predix-seed'...
remote: Counting objects: 13523, done.
remote: Compressing objects: 100% (15/15), done.
remote: Total 13523 (delta 6), reused 0 (delta 0), pack-reused 13508
Receiving objects: 100% (13523/13523), 46.62 MiB | 2.52 MiB/s, done.
Resolving deltas: 100% (5256/5256), done.
Checking connectivity... done.

    The command also creates a predix-seed directory which holds all of the Dashboard Seed project files.

    Record the pathname of the predix-seed directory for later use.

  3. Change to the predix-seed project directory:
    cd predix-seed
  4. Install the Dashboard Seed dependencies for these utilities: 
    npm install
bower install

    The package.json file lists the buildtime dependencies.  The bower.json file lists the runtime dependencies.

  5. Verify that the Dashboard Seed project files have been successfully installed:
    grunt serve

    A login dialog displays in a web browser.

See Deploying the Dashboard Seed to Cloud Foundry to modify the Dashboard Seed for deployment as a basic web application.
