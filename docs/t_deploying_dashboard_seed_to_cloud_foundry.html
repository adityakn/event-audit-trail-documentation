Deploying the Dashboard Seed to Cloud Foundry

Follow these steps to deploy the Dashboard Seed to Cloud Foundry as a basic web application that displays sample data after login:
  1. Set up required service instances for Cloud Foundry deployment. See Setting Up Required Services to Deploy the Dashboard Seed.
  2. Set up an Oauth2 client for user access to the Dashboard Seed in a web brow.ser. See Creating an OAuth2 Client to Access the Dashboard Seed)
  3. Test the Dashboard Seed in the Predix developer environment. See Testing the Dashboard Seed in the Predix Developer Environment.
  4. Add sample data to the View service instance asssociated with the Dashboard See.d See Adding Sample Data to the View Service Instance.
  5. Update the manifest.yml file. See Updating the Manifest File.
  6. Update the nginx.conf file. See Updating the NGINX Configuration.
  7. Push the Dashboard Seed to Cloud Foundry. See Pushing the Dashboard Seed to Cloud Foundry.
