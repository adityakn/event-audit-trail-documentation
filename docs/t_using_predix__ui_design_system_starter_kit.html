Using the Predix UI Design System Starter Kit

  • Locate the px-starter--kit-design repository in the PredixDev public GitHub organization
  • Find the application project directory <app_project> where the px-starter-kit-design files will be installed
  • Install the grunt task-runner utility for npm. See https://www.npmjs.com/package/grunt for more information.
  • Add the autoprefixer utility to grunt to parses CSS and adds vendor prefixes to CSS rules. See https://github.com/postcss/autoprefixer for more information
Note: See https://github.com/PredixDev/px-getting-started for more information on using the Predix UI design system files stored on the PredixDev public GitHub organization.

Follow these steps to use the Predix UI design system starter kit files:

  1. Install the grunt task-runner utility 
    npm install grunt -g
  2. Clone a copy of the px-starter-kit project files. For example: 
    git clone https://github.com/PredixDev/px-starter-kit-design.git
    A px-starter-kit-design directory is created that holds a tasks/options directory and configuration files.
  3. Change to the <app_proj> project directory:
    cd <app_proj>
  4. Create a tasks/options folder to include configuration files.
    mkdir ./tasks/options
  5. Copy the px-starter-kit-design/tasks/options configuration files to the application project directory:
    cp ../tasks/options/* <app_project>/tasks/options/.
  6. Update the autoprefixer.js and SASS.js files by replacing TODO with the correct file location
  7. Create a grunt configuration file: (gruntfile.js) using one of the following approaches:
    • Contains internal tasks.
    • Loops through all of the files in a specified folder.
    Note: See http://gruntjs.com/sample-gruntfile for more information about creating a grunt configuration file.

    The following gruntfile.js example loops through all the configuration files stored in the tasks/options directory.

    /**
 * @description Gruntfile that contains tasks to build, test and deploy a project.
 *
 * Configure all your tasks in the /tasks folder.
 *
 * For more help, see: http://gruntjs.com/getting-started
 */
module.exports = function(grunt) {
    'use strict';

    function loadConfig(path) {
        var glob = require('glob');
        var object = {};
        var key;

        glob.sync('*', {cwd: path}).forEach(function(option) {
           key = option.replace(/\.js$/, '');
            object[key] = require(path + option);
        });

        return object;
    }

    // Initial config
    var config = {
        pkg: grunt.file.readJSON('package.json')
    };

    // Load tasks from tasks folder
    grunt.loadTasks('tasks');

    grunt.util._.extend(config, loadConfig('./tasks/options/'));

    grunt.initConfig(config);

    require('load-grunt-tasks')(grunt);
};
