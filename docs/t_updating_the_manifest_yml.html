Updating the Manifest File

You need the pathname of the predix-seed directory where the Dashboard Seed project files are located. See Installing the Dashboard Seed.

You need the following values for required servcies that you recorded in Setting Up Required Services for Polymer Seed Authentication

  • Redis service instance name.
  • View service instance name.
  • UAA service instance uri.

You need to update the following values in the manifest.yml file before you can push the Dashboard Seed to Cloud Foundry:

  • my-predix-seed
  • your_redis_instance
  • your_view_service_instance
  • https://your-uaa-instance.grc-apps.svc.ice.ge.com
  • redis
  1. Change to the predix-seed project directory:
    cd predix-seed
  2. Change the placeholder value my-predix-seed to your application name app_name.
    Record the app_name for later use.
  3. Change the placeholder value your_redis_instance to your Redis service instanceID.
  4. Change the placeholder value your_view_service_instance to your View service instance name.
  5. Change the placeholder value https://your-uaa-instance.grc-apps.svc.ice.ge.com to uaa_instance_uri.
  6. Change the placeholder value redis to the Redis service in your marketplace.
    Note: Use the Cloud Foundry CLI to find the specific Redis service:
    cf marketplace

See Updating the NGINX Configuration before you push the Dashboard Seed to Cloud Foundry.
