Predix UI Components

Predix UI includes a set of components in the PredixDev public GitHub organization that are designed to achieve more complex user interactions than with Predix UI Basics alone when creating the user interface of a web application.

As a developer, you can use the code in the following Predix UI component repositories to produce these interactions.

Table 1. Predix UI Components
Repository Description Documentation
px-alert-message Displays alert information to a user. API Reference
px-app-nav Navigation bar or drawer for applications. API Reference
px-calendar-picker Component that allows the user to select a date or date range. API Reference
px-chart Component that defines a chart, using sub-elements for axis, series and control configuration. API Reference
px-chart-series-bar Component that defines a chart series that is displayed as a bar. API Reference
px-chart-series-histogram Component that defines a chart series that is displayed as a histogram. API Reference
px-chart-series-line Component that defines a chart series that is displayed as a line. API Reference
px-chart-series-scatter Component that defines a chart series that is displayed as a scatter. API Reference
px-chart-yaxis Component used for y-axis configuration for a chart. API Reference
px-clipboard Component that copies a value into the user's computer API Reference
px-context-browser Content browser displaying multiple levels of hierarchy for the current item. API Reference
px-data-table Data table with the expected data format. The data is passed as a JSON array. API Reference
px-datetime-field Component that allows the user to select a date and time precisely to the second or millisecond. API Reference
px-datetime-picker Component that allows the user to select a date and time precisely to the second or millisecond. API Reference
px-datetime-range-field Component that allows the user to select a date and time range precisely to the second or millisecond. API Reference
px-datetime-range-panel Component that allows the user to select a date and time range precisely to the second or millisecond. The main use of this component is to be used in the px-rangepicker. API Reference
px-deck-selector Drop-down display of available decks that returns the URL for a selected deck. API Reference
px-dropdown Component that opens a customizable dropdown list. API Reference
px-modal Component that indicates user interaction workflow using a modal. API Reference
px-overlay Component that provides an overlay for use with other components. API Reference
px-percent-circle Component that provides the simplest possible donut chart - a single ring that fills clockwise based on a percentage. API Reference
px-rangepicker Sets a range of date and time using a calendar-based user interface. API Reference
px-sample-cards Examples which demonstrate how to make cards. Examples
px-simple-bar-chart Displays a series of numeric values as a bar chart, or multiple series of numeric values as a stacked bar chart. API Reference
px-simple-horizontal-bar-chart Displays a series of numeric values as a horizontally-oriented bar chart. API Reference
px-simple-line-chart Displays a series of numeric values as a line chart. API Reference
px-slider Defines a value in a range or a range of values within set boundaries. API Reference
px-vis Set of base components to provide the ability to create charts. API Reference
px-vis-pie-chart Component to draw a pie or donut chart for some data. API Reference
px-vis-timeseries Component to create an interactive timeseries chart. API Reference
px-vis-xy-chart Component to draw lines or scatter against an X and Y axis. API Reference
px-widget-cards Creates cards with two, three, four or six components laid out in a grid. Examples
Note:

The API reference documentation for individual repositories also includes usage instructions and examples.

The README file for individual repositories tracks known limitations and restrictions as GitHub issues. File a GitHub issue if you encounter a problem using a Predix UI repository. For more information on filing a GitHub issue, see https://guides.github.com/features/issues/.
