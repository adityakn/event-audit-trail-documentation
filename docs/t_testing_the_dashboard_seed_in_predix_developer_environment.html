Testing the Dashboard Seed in the Predix Developer Environment

You need the following information to test the Dashboard Seed in the Predix developer environment

You need the pathname of the predix-seed directory where the Dashboard Seed project files are located (see Installing the Dashboard Seed).

  • Pathname of the predix-seed directory where the Dashboard Seed project files are located (see Installing the Dashboard Seed).
  • UAA instance uri that you recorded in Setting Up Required Services to Deploy the Dashboard Seed.
  • View service instanceId and uri that you recorded in Setting Up Required Services to Deploy the Dashboard Seed.

  • OAuth2 client name oauth2_client_name and base64ClientCredential that you recorded in Updating the OAuth2 Client for the Polymer Seed.

  • The user name and password of my-user that you added in Updating the OAuth2 Client for the Polymer Seed.

This procedure tests the Dashboard Seed in the Predix developer environment with the grunt serve plugin that starts a web server to run tasks and serve files. This web server can be accessed by loading the page http://localhost:9000/. See https://www.npmjs.com/package/grunt-serve for more information.

You need to record the authorization token for use when deploying the Dashboard Seed to Cloud Foundry:

  1. Change to the predix-seed/tasks/options/ directory.
    cd predix-seed/tasks/options/
  2. Modify the following objects in the connect.js file:
    1. Update the uaa object:
      uaa: {
    clientId: '<your-client>', 
    serverUrl: '<your-server-url>', 
    defaultClientRoute: '/about',
    base64ClientCredential: '<your-client-credential>'
},
      where:
      • <your-client>: oauth2_clientID
      • <your-server-url>: UAA instance uri
      • <your-client-credential>: base64ClientCredential
    2. Update the view-service proxy object:
      proxy: {
'/api/view-service(.*)': {
  url: '<your-view-service-url>/v1$1',
  instanceId: '<your-view-service-instanceID>'
}
      where:
      • url: the URL for your View service uri appended with "/v1$1" (for example: http://my-view-service.predix.io/v1$1).
      • instanceId: your View service instanceID.
  3. Run the Dashboard Seed in the Predix developer environment:
    grunt serve
  4. Log into the Dashboard Seed with the my-user user name and password that you added in Creating an OAuth2 Client to Access the Dashboard Seed.
    Record the authorization token displayed in the terminal after you run the grunt command and log into the Dashboard Seed. For example:
    authorization:'<token>'
