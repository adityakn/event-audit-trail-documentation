Pushing the Dashboard Seed to Cloud Foundry

You need the pathname of the predix-seed directory where the Dashboard Seed project files are located. See Installing the Dashboard Seed).

You need the my-user user name and password that you added in Creating an OAuth2 Client to Access the Dashboard Seed.

Follow these steps to deploy the Dashboard Seed to Cloud Foundry as a basic web application:
  1. Change to the predix-seed project directory:
    cd predix-seed
  2. Use grunt to create a distribution version of the Dashboard Seed in the dist folder with the nginx configuration files.
    grunt dist
  3. Deploy the Dashboard Seed to Cloud Foundry:
    cf push
    Note: Run the grunt dist command every time during normal development before you perform a cf push operation to deploy a web application to Cloud Foundry.

    The command returns the URL for the Dashboard Seed.

  4. Enter the URL in a web browser.

    A login dialog displays in a web browser.

  5. Log into the Dashboard Seed with the my-user user name and password that you added in Creating an OAuth2 Client to Access the Dashboard Seed.

    The Dashboard Seed displays with sample data.

See Customizing Web Applications with Predix UI for information about taiiloring the Dashboard Seed to your specific requirements by using the Predix UI platform to customize the user interface to Predix data and services..
