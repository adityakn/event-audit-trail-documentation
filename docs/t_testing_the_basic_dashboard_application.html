Testing the Basic Dashboard Application

You need the app_name that you recorded in Updating the Manifest File.

This procedure uses these developer tools to deploy the basic dashboard application to Cloud Foundry as a test.:
  • Node.js npm package management utility (see Installing the Dashboard Seed).
  • grunt command line interface for npm (see Installing the Dashboard Seed).
  • grunt dist plugin to create a distribution version of a web application (see Installing the Dashboard Seed).
  1. Enter these commands from the predix-seed directory:
    grunt dist
cf push
    Note: Run the grunt dist command every time during normal development to create a distribution version before performing the cf push operation to deploy a web application to Cloud Foundry.

    The command returns the URL for the basic dashboard application.

  2. Enter the URL in a web browser.

    A login dialog displays in a web browser.
