Predix UI Design System Overview

The Predix UI platform includes a UX framework that combine the following elements:
  • Components — User-interface element that allows you to add features or access services following consistent UX practices, standards and specification.
  • Cards — Container that resides within the content area of the user interface. You can create interactions between components within an individual card, as well as interactions with other cards based on user context.
  • Decks — Container that contains cards and components arranged in a specified order.
  • Views — Visual summary of the information displayed in a web application. It consists of one or more decks that an application user can select at runtime to quickly digest information and take appropriate action (see About the View Service for more information).

As a developer, Predix UI includes a design system in the PredixDev public GitHub organization to create intricate user interactions in a web application for your specific requiements. The Predix UI design system uses inuit.css as a CSS framework to provide the following benefits:

  • Web applications that follow consistent UX practices, standards, and specifications.
  • Responsive web design that provides an optimal viewing and interaction experience across laptop, tablet, phone, or large-screen devices.

You can consume as many Predix UI design system libraries as required to configure the user interface to Predix data and services in a web application. The system also includes Predix UI collections organized by common functions (such as lists or buttons) for convenient use when configuring a web application.

The modular architecture includes defined variables and default variants that provide the flexibility to change and configure existing applications, rather than use a large, complex, single-component repository:

Defined Variables

You can configure any Predix UI design system library or inuitcss module by defining any of its variables immediately before the @import declaration. For example: 
$inuit-btn-background:      rgb(247,247,247);
$inuit-btn-color:           rgb(47,47,47);
$inuit-enable-btn--full:    true;
@import "px-buttons-design/_objects.buttons.scss";

$inuit-enable-layout--middle:   true;
@import "px-layout-design/_objects.layout.scss";

$inuit-enable-flag--rev:        true;
$inuit-enable-flag--responsive: true;
@import "px-flag-design/_objects.flag.scss";

This method of modifying the Predix UI design system eliminates the need to edit any files directly (making it easier to update the framework), and also negates your need to configure an entire library from a large variable file.

Default Flags

To reduce the amount of code in your project, all Predix UI design system library features are disabled by default.

To turn features on, set their switches to true (again, before you import the file). This reduces the size of the Predix UI design system by enabling only the design libraries that you need. For example:
$inuit-enable-btn--full:    true;
$inuit-enable-btn--large:   true;
@import "px-buttons-design/_objects.buttons.scss";

Import Order

The order in which a Predix UI design system library or inuitcss module is imported affects the system operations:
  1. Settings – Global variables, site-wide settings, or configuration switches.
  2. Tools – Site-wide mixins and functions.
  3. Generic – Low-specificity, far-reaching rulesets (such as normalize.css).
  4. Base – Unclassed HTML elements (such as block quote {}or address {}).
  5. Objects – Objects, abstractions, and design patterns (such as .media {}).
  6. Components – Discrete, complete chunks of UI (such as .carousel {}).
  7. Trumps – High-specificity, very explicit selectors. Overrides and helper classes (such as .hidden {}).
For example:
// Settings
@import "px-colors-design/_settings.colors.scss";

// Generic
@import "px-normalize-design/_generic.normalize.scss";
@import "px-box-sizing-design/_generic.box-sizing.scss";
@import "px-helpers-design/_generic.helpers.scss";

// Base
@import "px-flexbox-design/_base.flexbox.scss";
@import "px-viewport-design/_base.viewport.scss";
@import "px-typography-design/_base.typography.scss";

// Trumps
@import "inuit-clearfix/_trumps.clearfix.scss";
@import "px-spacing-responsive-design/_trumps.spacing-responsive.scss";
@import "px-widths-responsive-design/_trumps.widths-responsive.scss";
Note: See https://github.com/PredixDev/px-getting-started#import-order for more information.
