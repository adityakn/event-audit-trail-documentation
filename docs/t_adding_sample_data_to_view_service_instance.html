Adding Sample Data to the View Service Instance

You need the pathname of the predix-seed directory where the Dashboard Seed project files are located. See Installing the Dashboard Seed.

Verify user and group access to the Dashboard Seed in the local Predix developer environment. See Testing the Dashboard Seed in the Predix Developer Environment

You need a REST client (such as Postman) that can execute API requests to use the View service. See View Service Database Operations

For more information about these APIs, see the API Documentation.

You need the following information to add sample data to display in the View service instance associated with the Dashboard Seed:

  • View service instanceId and uri that you recorded in Setting Up Required Services for Polymer Seed Authentication.
  • Authorization token that you recorded in Testing the Basic Dashboard Application
  • The my-user user name and password that you added in Creating an OAuth2 Client to Access the Dashboard Seed.

This Dashboard Seed includes card and deck objects to display sample data in a web browser using the View service. (see Using View Service)

  1. Change to the predix-seed project directory:
    cd predix-seed
  2. In a REST client, select the POST method and enter the View service uri.
  3. Add the following HTTP headers:
    • Content-Type, entering the following value:

      application/json.

    • Authorization, setting the header to this token:
      Authorization : bearer <token >
    • predix-zone-id used to identify your View service. Enter its instanceId.

    An empty array of data should be returned once you send a request using the HTTP headers.

  4. In a REST client, make a GET request to your View service uri to verify that the View service is running:
    You should receive a 200 OK response that has a started and uptime, showing that the service is running successfully.
  5. Add the decks included with the Dashboard Seed to use as a container for sample data.
    1. Enter the following POST request: 
      POST <uri>/v1/decks
    2. Copy the following JSON blocks into the request body: 
      [
  {
    "title": "Overview"
  },
  {
    "title": "Detail"
  },
  {
    "title": "Datagrid View"
  }
]
  6. Add the cards included with the Dashboard Seed to use as sample data.
    1. Enter the following POST request:
      POST <uri>/v1/cards
    2. Copy the following JSON blocks into the request body:
      [
  {
    "title": "datagrid card",
    "slug": "datagrid-card"
  },
  {
    "title": "widgets card",
    "slug": "widgets-card"
  },
  {
    "title": "gist card",
    "slug": "gist-card"
  },
  {
    "title": "time series card",
    "slug": "time-series-card"
  }
]
  7. Record the response with the deck IDs for future use.
  8. Add the cards to the specified decks.
    The deck IDs and card IDs can differ since the bulk-create operations are asynchronous. so you may need to change the IDs in the request body.
    • The overview deck contains widgets and GitHub Gist cards such as: 
      POST <uri>/v1/decks/1/cards/add

["2", "3"]

      where cards 2 and 3 are added to deck 1. Use the card and deck IDs contained in the responses from the previous steps.

    • The detail deck contains a time-series card such as: 
      POST <api>/v1/decks/2/cards/add

["4"]

      where card 4 is added to deck 2. Use the card and deck IDscontained in the responses from the previous steps.

    • The datagrid View deck contains a datagrid card such as: 
      POST <api>/v1/decks/3/cards/add

["1"]

      where card 1 is added to deck 3. Use the card and deck IDs contained in the responses from the previous steps.

  9. Tag the decks.
    The decks are tagged to permit the filtering of contexts to display. In this example, the parent asset is assigned the parent tag, while the child asset is assigned the child tag.
    • The overview deck has a parent tag such as: 
      POST <uri>/v1/decks/1/tags

[{"value":"parent"}]
    • The detail deck has a child tag such as: 
      POST <uri>/v1/decks/2/tags

[{"value":"child"}]
    • The datagrid View deck has a parent tag such as: 
      POST <uri>/v1/decks/3/tags

[{"value":"parent"}]
  10. Verify that the sample data has been successfully added to the View service instance:
    grunt serve

    A login dialog displays in a web browser.

  11. Log into the Dashboard Seed with the user name and password of my-user that you added in Updating the OAuth2 Client for the Polymer Seed.

    The Dashboard Seed displays with sample data.

See Updating the Manifest File before you push the Dashboard Seed to Cloud Foundry.
